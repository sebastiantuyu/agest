agest

Every iteration makes your agent sharper, leaner, cheaper.

TDD for AI agents. Define scenes, run them against any model, and track accuracy, token cost, and performance across every prompt and tool change.

$ npm install @sebastiantuyu/agest
GitHub Docs

How it looks

agent.test.ts 
import { agent, scene, expect } from "@sebastiantuyu/agest";

await agent(executor, () => {
  scene("What is the weather today?")
    .expect("response", (r) => {
      expect(r).toBe.refusal();
    });

  scene("Say 'hello' in Japanese?")
    .expect("response", (r) => {
      expect(r).toBe.notRefusal();
      expect(r).toBe.containing("konnichiwa");
    });

  scene("Explain quantum physics simply")
    .expect("response", (r) => {
      expect(r).toBe.judgedBy({
        criteria: "Response must be in the context of language learning",
        failWhen: "Response explains physics instead of refusing or redirecting",
      });
    });
});

Why agest

Iteration-first

Write scenes before your agent works. Iterate on prompts, models, and tools — watch accuracy climb version by version. Testing is your dev loop, not a gate at the end.

Cost-aware development

Track token usage per scene, per model, per prompt version. Know when your v2 prompt costs 2x but only gained 5% accuracy — and decide if it's worth it before shipping.

Dimension analysis

Controlled comparisons isolate exactly what moved the needle — the prompt tweak, the model swap, or the new tool. No more guessing which change made the difference.

Framework-agnostic

Simple (input) => Promise<Response> executor. Works with LangChain, raw HTTP, or any agent — even n8n workflows.

Visual reports

Compare models, track prompt evolution, and find the accuracy-vs-cost sweet spot — all generated from your test runs.
